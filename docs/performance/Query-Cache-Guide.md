## 🔄 缓存架构

### 查询缓存流程

```
┌─────────────────────────────────────────┐
│  用户查询                                │
│  - GET /api/v1/devices                   │
│  - GET /api/v1/telemetry                 │
└─────────────────────────────────────────┘
          ↓
┌─────────────────────────────────────────┐
│  查询缓存中间件                          │
│  - 检查缓存 Key                          │
│  - 命中 -> 返回缓存结果                  │
│  - 未命中 -> 继续查询                    │
└─────────────────────────────────────────┘
          ↓ (未命中)
┌─────────────────────────────────────────┐
│  数据库查询                              │
│  - 执行 SQL 查询                         │
│  - 获取查询结果                          │
└─────────────────────────────────────────┘
          ↓
┌─────────────────────────────────────────┐
│  缓存结果                                │
│  - 生成缓存 Key                          │
│  - 存储到 Redis                          │
│  - 设置 TTL                              │
└─────────────────────────────────────────┘
          ↓
┌─────────────────────────────────────────┐
│  返回结果                                │
│  - 返回查询结果                          │
│  - 记录缓存统计                          │
└─────────────────────────────────────────┘
```

## 🔧 缓存策略

### Cache-Aside 策略

```
读取流程:
1. 检查缓存
2. 缓存命中 -> 返回结果
3. 缓存未命中 -> 查询数据库 -> 缓存结果 -> 返回

写入流程:
1. 更新数据库
2. 删除缓存 (下次读取时重新加载)
```

### 缓存 Key 设计

```
缓存 Key 格式:
cache:{tenant_id}:{resource}:{query_hash}

示例:
- cache:tenant-001:devices:list:abc123
- cache:tenant-001:device:device-001:def456
- cache:tenant-001:telemetry:device-001:ghi789
```
